sfq (Salesforce Query)

sfq is a lightweight Python wrapper library designed to simplify querying Salesforce, reducing repetitive code for accessing Salesforce data.

For more varied workflows, consider using an alternative like Simple Salesforce. This library was even referenced on the Salesforce Developers Blog.

Features

  • Simplified query execution for Salesforce instances.
  • Integration with Salesforce authentication via refresh tokens.
  • Option to interact with Salesforce Tooling API for more advanced queries.

Installation

You can install the sfq library using pip:

pip install sfq

Usage

Library Querying

from sfq import SFAuth

# Initialize the SFAuth class with authentication details
sf = SFAuth(
    instance_url="https://example-dev-ed.trailblaze.my.salesforce.com",
    client_id="your-client-id-here",
    client_secret="your-client-secret-here",
    refresh_token="your-refresh-token-here"
)

# Execute a query to fetch account records
print(sf.query("SELECT Id FROM Account LIMIT 5"))

# Execute a query to fetch Tooling API data
print(sf.tooling_query("SELECT Id, FullName, Metadata FROM SandboxSettings LIMIT 5"))

How to Obtain Salesforce Tokens

To use the sfq library, you'll need a client ID and refresh token. The easiest way to obtain these is by using the Salesforce CLI:

Steps to Get Tokens

  1. Install the Salesforce CLI:
    Follow the instructions on the Salesforce CLI installation page.

  2. Authenticate with Salesforce:
    Login to your Salesforce org using the following command:

    sf org login web --alias int --instance-url https://corpa--int.sandbox.my.salesforce.com

  3. Display Org Details:
    To get the client ID, client secret, refresh token, and instance URL, run:

    sf org display --target-org int --verbose --json

    The output will look like this:

    {

  "result": {
    "id": "00Daa0000000000000",
    "apiVersion": "63.0",
    "accessToken": "00Daa0000000000000!evaU3fYZEWGUrqI5rMtaS8KYbHfeqA7YWzMgKToOB43Jk0kj7LtiWCbJaj4owPFQ7CqpXPAGX1RDCHblfW9t8cNOCNRFng3o",
    "instanceUrl": "https://example-dev-ed.trailblaze.my.salesforce.com",
    "username": "user@example.com",
    "clientId": "PlatformCLI",

  4. Extract and Use the Tokens:
    The sfdxAuthUrl is structured as:

    force://<client_id>:<client_secret>:<refresh_token>@<instance_url>

    This means with the above output sample, you would use the following information:

    instance_url=instance_url,
    client_id=client_id,
    client_secret=client_secret,
    refresh_token=refresh_token,
)

Important Considerations

  • Security: Safeguard your client_id, client_secret, and refresh_token diligently, as they provide access to your Salesforce environment. Avoid sharing or exposing them in unsecured locations.
  • Efficient Data Retrieval: The query and cquery function automatically handles pagination, simplifying record retrieval across large datasets. It's recommended to use the LIMIT clause in queries to control the volume of data returned.
  • Advanced Tooling Queries: Utilize the tooling_query function to access the Salesforce Tooling API. This option is designed for performing complex operations, enhancing your data management capabilities.

 306    def _prepare_payload(self) -> Dict[str, Optional[str]]:
 307        """
 308        Prepare the payload for the token request.
 309

 398    def _new_token_request(self, payload: Dict[str, str]) -> Optional[Dict[str, Any]]:
 399        """
 400        Perform a new token request using the provided payload.
 401
 402        :param payload: Payload for the token request.
 403        :return: Parsed JSON response or None on failure.
 404        """
 405        headers = self._get_common_headers(recursive_call=True)
 406        headers["Content-Type"] = "application/x-www-form-urlencoded"
 407        del headers["Authorization"]
 408
 409        body = "&".join(f"{key}={quote(str(value))}" for key, value in payload.items())
 410        status, data = self._send_request("POST", self.token_endpoint, headers, body)
 411
 412        if status == 200:
 413            logger.trace("Token refresh successful.")
 414            return json.loads(data)
 415
 416        if status:
 417            logger.error("Token refresh failed: %s", status)
 418            logger.debug("Response body: %s", data)
 419
 420        return None

 422    def _http_resp_header_logic(self, response: http.client.HTTPResponse) -> None:
 423        """
 424        Perform additional logic based on the HTTP response headers.
 425
 426        :param response: The HTTP response object.
 427        :return: None
 428        """
 429        logger.trace(
 430            "Response status: %s, reason: %s", response.status, response.reason
 431        )
 432        headers = response.getheaders()
 433        headers_list = [(k, v) for k, v in headers if not v.startswith("BrowserId=")]
 434        logger.trace("Response headers: %s", headers_list)
 435        for key, value in headers_list:
 436            if key == "Sforce-Limit-Info":
 437                current_api_calls = int(value.split("=")[1].split("/")[0])
 438                maximum_api_calls = int(value.split("=")[1].split("/")[1])
 439                usage_percentage = round(current_api_calls / maximum_api_calls * 100, 2)
 440                if usage_percentage > self._high_api_usage_threshold:
 441                    logger.warning(
 442                        "High API usage: %s/%s (%s%%)",
 443                        current_api_calls,
 444                        maximum_api_calls,
 445                        usage_percentage,
 446                    )
 447                else:
 448                    logger.debug(
 449                        "API usage: %s/%s (%s%%)",
 450                        current_api_calls,
 451                        maximum_api_calls,
 452                        usage_percentage,
 453                    )

 455    def _refresh_token_if_needed(self) -> Optional[str]:
 456        """
 457        Automatically refresh the access token if it has expired or is missing.
 458
 459        :return: A valid access token or None if refresh failed.
 460        """
 461        if self.access_token and not self._is_token_expired():
 462            return self.access_token
 463
 464        logger.trace("Access token expired or missing, refreshing...")
 465        payload = self._prepare_payload()
 466        token_data = self._new_token_request(payload)
 467
 468        if token_data:
 469            self.access_token = token_data.get("access_token")
 470            issued_at = token_data.get("issued_at")
 471
 472            try:
 473                self.org_id = token_data.get("id").split("/")[4]
 474                self.user_id = token_data.get("id").split("/")[5]
 475                logger.trace(
 476                    "Authenticated as user %s for org %s (%s)",
 477                    self.user_id,
 478                    self.org_id,
 479                    token_data.get("instance_url"),
 480                )
 481            except (IndexError, KeyError):
 482                logger.error("Failed to extract org/user IDs from token response.")
 483
 484            if self.access_token and issued_at:
 485                self.token_expiration_time = int(issued_at) + self.token_lifetime
 486                logger.trace("New token expires at %s", self.token_expiration_time)
 487                return self.access_token
 488
 489        logger.error("Failed to obtain access token.")
 490        return None

 657    def _paginate_query_result(self, initial_result: dict, headers: dict) -> dict:
 658        """
 659        Helper to paginate Salesforce query results (for both query and cquery).
 660        Returns a dict with all records combined.
 661        """
 662        records = list(initial_result.get("records", []))
 663        done = initial_result.get("done", True)
 664        next_url = initial_result.get("nextRecordsUrl")
 665        total_size = initial_result.get("totalSize", len(records))
 666
 667        while not done and next_url:
 668            status_code, data = self._send_request(
 669                method="GET",
 670                endpoint=next_url,
 671                headers=headers,
 672            )
 673            if status_code == 200:
 674                next_result = json.loads(data)
 675                records.extend(next_result.get("records", []))
 676                done = next_result.get("done", True)
 677                next_url = next_result.get("nextRecordsUrl")
 678                total_size = next_result.get("totalSize", total_size)
 679            else:
 680                logger.error("Failed to fetch next records: %s", data)
 681                break
 682
 683        paginated = dict(initial_result)
 684        paginated["records"] = records
 685        paginated["done"] = done
 686        paginated["totalSize"] = total_size
 687        if "nextRecordsUrl" in paginated:
 688            del paginated["nextRecordsUrl"]
 689        return paginated

 803    def cquery(
 804        self, query_dict: dict[str, str], batch_size: int = 25, max_workers: int = None
 805    ) -> Optional[Dict[str, Any]]:
 806        """
 807        Execute multiple SOQL queries using the Composite Batch API with threading to reduce network overhead.
 808        The function returns a dictionary mapping the original keys to their corresponding batch response.
 809        The function requires a dictionary of SOQL queries with keys as logical names (referenceId) and values as SOQL queries.
 810        Each query (subrequest) is counted as a unique API request against Salesforce governance limits.
 811
 812        :param query_dict: A dictionary of SOQL queries with keys as logical names and values as SOQL queries.
 813        :param batch_size: The number of queries to include in each batch (default is 25).
 814        :param max_workers: The maximum number of threads to spawn for concurrent execution (default is None).
 815        :return: Dict mapping the original keys to their corresponding batch response or None on failure.
 816        """
 817        if not query_dict:
 818            logger.warning("No queries to execute.")
 819            return None
 820
 821        def _execute_batch(batch_keys, batch_queries):
 822            endpoint = f"/services/data/{self.api_version}/composite/batch"
 823            headers = self._get_common_headers()
 824
 825            payload = {
 826                "haltOnError": False,
 827                "batchRequests": [
 828                    {
 829                        "method": "GET",
 830                        "url": f"/services/data/{self.api_version}/query?q={quote(query)}",
 831                    }
 832                    for query in batch_queries
 833                ],
 834            }
 835
 836            status_code, data = self._send_request(
 837                method="POST",
 838                endpoint=endpoint,
 839                headers=headers,
 840                body=json.dumps(payload),
 841            )
 842
 843            batch_results = {}
 844            if status_code == 200:
 845                logger.debug("Composite query successful.")
 846                logger.trace("Composite query full response: %s", data)
 847                results = json.loads(data).get("results", [])
 848                for i, result in enumerate(results):
 849                    key = batch_keys[i]
 850                    if result.get("statusCode") == 200 and "result" in result:
 851                        paginated = self._paginate_query_result(
 852                            result["result"], headers
 853                        )
 854                        batch_results[key] = paginated
 855                    else:
 856                        logger.error("Query failed for key %s: %s", key, result)
 857                        batch_results[key] = result
 858            else:
 859                logger.error(
 860                    "Composite query failed with HTTP status %s (%s)",
 861                    status_code,
 862                    data,
 863                )
 864                for i, key in enumerate(batch_keys):
 865                    batch_results[key] = data
 866                logger.trace("Composite query response: %s", data)
 867
 868            return batch_results
 869
 870        keys = list(query_dict.keys())
 871        results_dict = OrderedDict()
 872
 873        with ThreadPoolExecutor(max_workers=max_workers) as executor:
 874            futures = []
 875            BATCH_SIZE = batch_size
 876            for i in range(0, len(keys), BATCH_SIZE):
 877                batch_keys = keys[i : i + BATCH_SIZE]
 878                batch_queries = [query_dict[key] for key in batch_keys]
 879                futures.append(
 880                    executor.submit(_execute_batch, batch_keys, batch_queries)
 881                )
 882
 883            for future in as_completed(futures):
 884                results_dict.update(future.result())
 885
 886        logger.trace("Composite query results: %s", results_dict)
 887        return results_dict

 891    ) -> Optional[Dict[str, Any]]:
 892        """
 893        Execute the Collections Delete API to delete multiple records using multithreading.

 895        :param ids: A list of record IDs to delete.

 904            endpoint = f"/services/data/{self.api_version}/composite/sobjects?ids={','.join(chunk)}&allOrNone=false"

 917                logger.error("Collections delete API request failed: %s", status_code)
 918                logger.debug("Response body: %s", resp_data)
 919                return None
 920

 923            futures = [executor.submit(delete_chunk, chunk) for chunk in chunks]

 940        """

 942
 943        :param update_dict: A dictionary of keys of records to be updated, and a dictionary of field-value pairs to be updated, with a special key '_' overriding the sObject type which is otherwise inferred from the key. Example:

 804    def cquery(
 805        self, query_dict: dict[str, str], batch_size: int = 25, max_workers: int = None
 806    ) -> Optional[Dict[str, Any]]:
 807        """
 808        Execute multiple SOQL queries using the Composite Batch API with threading to reduce network overhead.
 809        The function returns a dictionary mapping the original keys to their corresponding batch response.
 810        The function requires a dictionary of SOQL queries with keys as logical names (referenceId) and values as SOQL queries.
 811        Each query (subrequest) is counted as a unique API request against Salesforce governance limits.
 812
 813        :param query_dict: A dictionary of SOQL queries with keys as logical names and values as SOQL queries.
 814        :param batch_size: The number of queries to include in each batch (default is 25).
 815        :param max_workers: The maximum number of threads to spawn for concurrent execution (default is None).
 816        :return: Dict mapping the original keys to their corresponding batch response or None on failure.
 817        """
 818        if not query_dict:
 819            logger.warning("No queries to execute.")
 820            return None
 821
 822        def _execute_batch(batch_keys, batch_queries):
 823            endpoint = f"/services/data/{self.api_version}/composite/batch"
 824            headers = self._get_common_headers()
 825
 826            payload = {
 827                "haltOnError": False,
 828                "batchRequests": [
 829                    {
 830                        "method": "GET",
 831                        "url": f"/services/data/{self.api_version}/query?q={quote(query)}",
 832                    }
 833                    for query in batch_queries
 834                ],
 835            }
 836
 837            status_code, data = self._send_request(
 838                method="POST",
 839                endpoint=endpoint,
 840                headers=headers,
 841                body=json.dumps(payload),
 842            )
 843
 844            batch_results = {}
 845            if status_code == 200:
 846                logger.debug("Composite query successful.")
 847                logger.trace("Composite query full response: %s", data)
 848                results = json.loads(data).get("results", [])
 849                for i, result in enumerate(results):
 850                    key = batch_keys[i]
 851                    if result.get("statusCode") == 200 and "result" in result:
 852                        paginated = self._paginate_query_result(
 853                            result["result"], headers
 854                        )
 855                        batch_results[key] = paginated
 856                    else:
 857                        logger.error("Query failed for key %s: %s", key, result)
 858                        batch_results[key] = result
 859            else:
 860                logger.error(
 861                    "Composite query failed with HTTP status %s (%s)",
 862                    status_code,
 863                    data,
 864                )
 865                for i, key in enumerate(batch_keys):
 866                    batch_results[key] = data
 867                logger.trace("Composite query response: %s", data)
 868
 869            return batch_results
 870
 871        keys = list(query_dict.keys())
 872        results_dict = OrderedDict()
 873
 874        with ThreadPoolExecutor(max_workers=max_workers) as executor:
 875            futures = []
 876            BATCH_SIZE = batch_size
 877            for i in range(0, len(keys), BATCH_SIZE):
 878                batch_keys = keys[i : i + BATCH_SIZE]
 879                batch_queries = [query_dict[key] for key in batch_keys]
 880                futures.append(
 881                    executor.submit(_execute_batch, batch_keys, batch_queries)
 882                )
 883
 884            for future in as_completed(futures):
 885                results_dict.update(future.result())
 886
 887        logger.trace("Composite query results: %s", results_dict)
 888        return results_dict

824            headers = self._get_common_headers()

827                "haltOnError": False,

830                        "method": "GET",

832                    }
